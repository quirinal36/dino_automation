## 제1원칙 사고방식이란?

### 비유로 배우는 사고방식

**비유 사고 (Analogy Thinking)**
> "다른 게임 봇들이 딥러닝을 사용하니까, 우리도 딥러닝을 써야 해"

**제1원칙 사고 (First Principles Thinking)**
> "공룡이 점프하려면 정확히 무엇이 필요한가? 그것을 달성하는 가장 단순한 방법은?"

### 핵심 질문

제1원칙 사고의 핵심은 이 질문입니다:

> **"이것이 작동하려면 반드시 참이어야 하는 가장 기본적인 것들은 무엇인가?"**

## 문제 분해하기

### 원래 문제

> "Chrome Dino 게임을 자동으로 플레이하는 프로그램을 만든다"

### Step 1: 사람은 어떻게 게임을 하는가?

게임을 직접 플레이할 때 당신이 하는 행동을 분석해봅시다:

1. **눈**으로 화면을 본다
2. **뇌**로 장애물이 오는지 판단한다
3. **손가락**으로 스페이스바를 누른다

### Step 2: 컴퓨터로 번역하기

| 사람 | 컴퓨터 | 필요한 기술 |
|------|--------|------------|
| 눈으로 본다 | 화면을 캡처한다 | 스크린샷 |
| 뇌로 판단한다 | 이미지를 분석한다 | 이미지 처리 |
| 손가락으로 누른다 | 키보드 입력을 보낸다 | 키보드 자동화 |

### Step 3: 더 깊이 분해하기

**"이미지를 분석한다"를 더 분해하면:**

1. 어디를 볼 것인가? → **ROI (관심 영역) 설정**
2. 무엇을 찾을 것인가? → **장애물 (어두운 물체)**
3. 어떻게 찾을 것인가? → **밝기 비교**

**"장애물을 찾는다"를 더 분해하면:**

- 게임 배경: 흰색 (밝음)
- 장애물: 검은색/회색 (어두움)
- **결론**: 어두운 픽셀이 일정 비율 이상이면 장애물!

## 최소한의 해결책

제1원칙 사고의 결과, 우리에게 필요한 것:

### 필수 기능 5가지

```
1. 화면 캡처       → Pillow (ImageGrab)
2. 특정 영역 추출  → NumPy (배열 슬라이싱)
3. 밝기 분석       → NumPy + OpenCV
4. 키보드 입력     → PyAutoGUI
5. 설정 저장       → JSON
```

## 생각해보기

### 질문 1: 왜 머신러닝이 필요 없을까?

우리 문제에서 "장애물 인식"은 얼마나 복잡한가요?

- 선인장의 "모양"을 알아야 할까요?
- 새의 "종류"를 구분해야 할까요?
- 아니면 그냥 "뭔가 어두운 것이 있다"만 알면 될까요?

### 질문 2: 화면 전체를 분석해야 할까?

1920x1080 화면에서:
- 공룡은 왼쪽에 있음
- 장애물은 공룡 앞에서 다가옴
- 점수나 구름은 분석할 필요 없음

**그렇다면 어느 영역만 보면 될까요?**

### 질문 3: 얼마나 자주 화면을 봐야 할까?

- 1초에 1번? → 너무 느림 (장애물을 놓침)
- 1초에 1000번? → 너무 빠름 (불필요한 낭비)
- **적당한 빈도는?**

## 핵심 정리

### 제1원칙 사고의 3단계

1. **분해**: 문제를 가장 기본적인 요소로 나눈다
2. **검증**: 각 요소가 정말 필요한지 확인한다
3. **재구성**: 필요한 것만으로 해결책을 만든다

### 우리 프로젝트에 적용

| 단계 | 원래 생각 | 제1원칙 적용 후 |
|------|----------|----------------|
| 인식 | "AI가 선인장을 인식해야 해" | "어두운 픽셀 비율만 확인하면 돼" |
| 영역 | "화면 전체를 분석해야 해" | "공룡 앞 작은 영역만 보면 돼" |
| 반응 | "복잡한 패턴 매칭이 필요해" | "어두우면 점프, 끝" |
